Advantage


  1. - Select (or ' Add') a Credit Policy


  2. - Begins with PubCode or BillingOrg#


  3. - PubCode + 6-Character-Code (could be term|price|renewal)


  4. - Bundling the credit policy, choice, and offer information
    - Things you'll need to complete a 'Promotion Offer'
    • Unique Promotional Offer Code [16-Characters]
      • Begin with pub code or owning org [3-Chars]
      • If it's a cross-sell (then place an X in front of pub code/owning org) [4-Chars]
    • Description of offer
    • Publication Code of subscription being sold
    • Premium Set ID/code
    • Credit Policy ID/code
    • Owning Organization for affiliate

    • Then once the offer is setup, we should add choices
      • Item number - a pub code (XXX) or item number (###) in promotion
        (usually the first 3-chars of promotion offer code)
      • Billing currency of the offer
        (usually US$)
      • Promotion chocie - Subscription or item choice code
      • Be sure to add a description of the offer


  5. - Code based on specific fields
    Method/How [1-Char] CIRC/PROD CODE [3-Chars] YEAR/MONTH [Year - R(2015) | Month - 0-9A-C] UNIQUE ID



  6. - Same codes as the first 3 fields of the campaign code; Followed by 2 unique chars for ID purposes
    - If a campaign was recently created, some information for the promotions section will be autofilled
    - Fill out all pertient information, but especially the following:
    • List - for direct mail, web or any non-Milo list
    • Milo List - for Agora eletter or pub
    • List segment
    • Effective from - most likely the date of the day you make the promotion
    • Quantity mailed
    • Acquitision
      • UI: Web/Email Promotion Internal
      • UX: Web/Email Promotion External
      • UP: Paid Direct Mail Promo
      • US: Insert Promo

  • Choice code must match the choice code in Advantage

  • Under Cross Sell/Upsell, teh Yes/No dropdown determines the following:
    • Yes = The page taken to if the customer selects yes

    • No = The page taken to if they select no

  • Offer Name = URL_Nickname - Price1_Choice_Code

Efforts

This page serves as a quick reference to special things to note for Efforts in IRIS


  • First things first, you should schedule the date! - Error will appear if you don't!

  • Choose your Effort Destination & Item Type; Once you choose Item Type you'll get a dropdown to other fields

  • Once you select Effort Type additional dropdowns that require attention will appear
    Including: List | Target | Segment

  • For Host 1% you'll have to choose a percentage (format = 0.00) of revenue to be split with a partner

  • Host 2% only used in a 3-way split of

  • The Options that appear at the bottom of the page
    • Row Color - assigns an identifying color to teh row that will contain the effort info

    • Gauntlet - this is a gauntlet series effort

    • Testing Group - create testing groups that will both categorize and organize marketing test efforts and/or group special efforts together for easy access

    • Vanity Url - create a vanity redirect for any promotion

    • Lift Note Only - Displays lift note (and lift note only) in dedicated email. If not selected, will only show user designed content or - rarely - a full promotion

    • Order Form Only - Bypass primary content or promotional step; Defaults to row-color yellow

    • Recurring - Automatically make new efforts, and place them in 'MyQueue'

    • Abandon email - Select email creative to send customers who abandoned promotion

    • URL Fix - Ability to fix URL on live promotion (if there's a problem)

    • Opens Pixel - For automatic tracking of external mailings & internal promotions



  • The purpose of a multivariate is to VARY a singular feature of the effort (content page | order form | Confirmation page)

  • Keeps track of how each VARIATE is doing in terms of its marketing campaign

  • You can click 'Edit Multivariate' under the 'Effort Details page to Activate/Deactivate a paticular VARIATE

IRIS Practice Exercises

The following excercises are intended as practice to familiarize yourself with the IRIS user interface and workflow.


Exercise One

PRE-CONDITIONS: Go to the publishing services instance of IRIS (iris.pubsvs.com) and create a new offer for the item LD1.

Tasks:

  1. Create an offer in IRIS with 2 main prices / choices.

  2. Attach a cross sell offer to the offer in step 1. It can be any item other than LD1

  3. Create a piece of HTML content using the offer from step 1

  4. Create a piece of video content and attach the HTML version to that content

  5. No create a dedicated effort going to MC TEST LIST with the target IRIS_TEST_LIST-IRISUSER_1 using the version you just created. Complete the following:
    1. Enter an effort tag titled '[enter your name]'s First Test (trick question)
    2. Create and save a lift note from the effort level
    3. Enter a note on the effort level
    4. Create and assign an effort row color
    5. Create and assign the effort to a testing group
    6. Give yourself receive domain request email and paid code email permissions.

  6. Go to My Queue and request the promo code for the dedicated effort that you just created. Click on the link in the promo code request email and enter a code (it can be random)

  7. Go back to the effort, copy it and assign it to a multivariate group but do not add a split

  8. Do not add additional multivariate efforts

  9. Repeat step 7 with the new effort you created

  10. Now go to the mailing just created (you may not have known it but you just created a mailing)

  11. Send an IRIS test email to yourself!
NOTE: Your comments will not be saved if you exit/refresh the page, so be sure to copy & paste them elsewhere when you're done with the exercise.

Exercise Two

NOTE: Exercise 2 involves lead gen training. It will be more difficult than the first one because the lead gen application leaves little to be desired (and we know that lead gen in 4.0 will be completely different).


PRE-CONDITION: Go to the publishing services instance of IRIS (iris.pubsvs.com)

Tasks:

  1. This offer should include a cross sell for the list IRIS.

  2. Create a lead gen offer using the IRIS Test List and attach the co-reg from step 1.

  3. Create a signup page that also includes a checkbox for the coreg (I don't believe the system supports this through the WYSIWYG, you may need to code this yourself). I can find examples if you need them.

  4. Create a signup page that does not have a co-reg checkbox.

  5. Create a confirmation page that includes a co-reg signup form.

  6. Create a co-reg confirmation email

  7. Create a confirmation email

  8. Create a report (this may be out of sequence but the system should prompt you to create all of these entities).

  9. Now go to the effort level and create an effort using the signup page from step 3 using the confirmation page from step 5.

  10. Copy the effort created in step 9 and create a new effort but this time use the page from step 4. Select the co-reg confirmation page you created in step 5 and add this effort to the multivariate group.

  11. Go to my queue and request a promo code for those two efforts.
NOTE: Your comments will not be saved if you exit/refresh the page, so be sure to copy & paste them elsewhere when you're done with the exercise.
